基于前四轮对**操作指引、内容认知、资产管理、主动预警、个性化适配、流程复盘、情感陪伴、元认知控制、跨应用编排、非屏实体推理、预测性习惯**这十一大维度的深度挖掘，并严格遵循“不执行操作、不侵犯隐私、不跟踪动态视频”三条红线，现在我为您整理出**HAJIMI智能桌面指引助手完整、穷尽式的用户文本指令情形总览**。

这份整理分为四个层级：**核心意图域（用户到底想干什么）**、**指代与描述方式（用户怎么表达目标）**、**任务结构与交互模式（任务的复杂程度与触发方式）**、**系统绝对边界（哪些情形不予处理及如何回应）**。

### 第一层级：核心意图域（共九大类，涵盖所有“显性+隐性”诉求）

| 大类 | 细分情形（典型用户原话） | 系统处理本质（仅输出形式，无操作） |
| :--- | :--- | :--- |
| **一、操作执行指引** | **1.1 单步原子操作**：“帮我点‘确定’”、“怎么关闭这个标签页” | 输出唯一单步指令，配合箭头+高亮框指向单一元素。 |
| | **1.2 多步单应用流程**：“如何安装微信”、“怎么设置打印机” | 生成恒定步骤蓝图（如1~5步），逐步高亮当前需操作的元素，步骤列表展示全部。 |
| | **1.3 跨应用编排流程**：“把网页表格复制到Excel并保存到桌面” | 输出跨窗口接力步骤，在“切换窗口”处插入暂停锚点，并标注任务栏目标图标。 |
| | **1.4 带强约束条件的操作**：“下载软件但别装在C盘”、“千万别点那个广告” | 将约束转化为步骤中的“特别警示”（如用红字标出备选路径或禁止点击的区域）。 |
| **二、界面元素认知** | **2.1 静态图标/文字释义**：“这个盾牌图标是什么意思” | 输出概念解释文字，并高亮该图标位置（若有）。 |
| | **2.2 状态异常询问**：“为什么这个按钮是灰色的”、“这里怎么有个红叉” | 分析前置依赖（如“因未勾选协议”），用箭头指向依赖项并给出前置操作建议。 |
| **三、异常诊断与恢复** | **3.1 主动悔步/回退**：“上一步点错了怎么办”、“我想回到第三步” | 读取历史蓝图，标注出界面上的“返回/取消”按钮，告知用户手动点击几次可回退。 |
| | **3.2 被动卡住/报错**：“弹出了一个报错框，怎么处理”、“下一步没反应” | 解释报错含义，输出解决步骤；若屏幕指纹与蓝图不符，主动弹窗询问“跳过/重试/终止”。 |
| | **3.3 非屏实体诊断**：“后台有没有可疑进程”、“我电脑是不是有重复照片” | **声明能力盲区**，转而指引用户打开任务管理器或搜索框，并提供手动筛选的关键词建议。 |
| **四、界面导航与定位** | **4.1 静态路径查找**：“控制面板在哪”、“怎么打开任务管理器” | 输出右键/快捷键的组合步骤，并用箭头指向开始菜单或任务栏的入口。 |
| | **4.2 已开启窗口/后台定位**：“我刚刚打开的记事本跑哪去了” | 若窗口在任务栏，高亮任务栏对应图标；若在托盘，标注托盘展开区并语音提示位置。 |
| **五、内容认知与信息处理** | **5.1 屏幕归纳摘要**：“总结这个网页”、“提炼这封邮件的要点” | **禁用所有箭头/高亮**，仅在对话区输出结构化文字摘要+TTS播报。 |
| | **5.2 视觉元素描述**：“这张图/图表里画了什么”、“这个界面布局怎样” | 输出循序渐进的口述场景描述（适用于视障辅助或快速理解）。 |
| | **5.3 数据提取与比对**：“提取表格数据”、“对比这两个商品价格” | 以纯文本/Markdown表格输出提取结果，并附带免责声明（“识别可能有误”）。 |
| | **5.4 多语言翻译**：“翻译这段英文”、“这个德语词是什么意思” | 输出译文，并可在翻译后追问“是否要指引操作译文对应的按钮”。 |
| **六、桌面环境与资产管理** | **6.1 空间归类**：“把桌面这3个文档归到文件夹里” | 输出手动操作步骤（如“按住Ctrl多选→右键剪切→双击目标文件夹→粘贴”），并圈选所有目标文件。 |
| | **6.2 查找与定位**：“帮我找出下载文件夹里所有的PDF” | 指引用户打开下载文件夹并手动在搜索框输入“*.pdf”，不主动遍历硬盘。 |
| **七、状态监控与主动预警** | **7.1 进度节点提醒**：“下载完成时提醒我” | 后台低频率轮询进度条/状态文字，满足条件时在挂件角落弹出轻量气泡通知。 |
| | **7.2 错误预防**：“如果C盘快满了就警告我” | 检测到容量提示文字时，主动语音/弹窗预警，并标注“清理磁盘”按钮的位置。 |
| | **7.3 习惯防御**：“我总忘取消附加软件，下次帮我提前标出来” | 在后续安装类蓝图中固定插入前置警示步骤，用闪烁红框标出需取消的勾选项。 |
| **八、流程复盘与教学记忆** | **8.1 导出操作教程**：“把刚才的步骤生成图文教程保存” | 在对话区输出纯文本/Markdown步骤清单，并指引用户手动复制粘贴到记事本。 |
| | **8.2 操作复述与校验**：“把我刚才的动作复述一遍，看我做得对吗” | 读取上下文历史中的蓝图执行记录，回放已完成的步骤链。 |
| | **8.3 命名与固化个人模板**：“存为‘给爸妈的WiFi教程’” | 将当前蓝图存入用户级本地缓存，下次同类提问优先调用此模板并注明来源。 |
| **九、情感陪伴与语境缓解** | **9.1 挫败宣泄回应**：“哎呀烦死了，又报错” | 先进行心理缓冲（“别着急，不伤电脑”），再输出**极简单步指令**并降低信息密度。 |
| | **9.2 恐惧安全兜底**：“我不敢点了，怕删错” | 先高亮显示“取消/关闭”按钮作为安全退路，待用户冷静后再输出正式指引。 |

### 第二层级：指代与描述方式（跨所有意图域的“目标定位”情形）

无论用户属于上述哪个大类，他们对“操作对象”的描述方式，决定了系统如何定位屏幕元素：

| 描述方式 | 典型原话 | 系统处理响应逻辑 |
| :--- | :--- | :--- |
| **显式命名** | “点击‘确定’按钮” | 直接OCR匹配文本+元素类型，精准定位。若有两个同名，触发二选一澄清。 |
| **视觉位置** | “左上角那个蓝色的按钮” | 结合坐标（bbox）进行空间推理匹配。若无元素，主动联想邻近元素并提议。 |
| **指示代词（依赖焦点）** | “点这个”、“选那个” | **强制依赖**鼠标悬停位置或键盘焦点（光标所在处）。若两者皆无，**必须触发主动澄清**（“请将鼠标移到目标上再问”）。 |
| **模糊/口语化描述** | “那个圆圆的像齿轮的东西” | 将描述传给多模态LLM进行视觉语义匹配，返回Top3候选ID，反问用户确认。 |
| **上下文接力指代** | （上句刚翻译完德语）“好，点一下它” | 系统自动将“它”映射为前一句翻译结果所对应的屏幕元素ID，无缝切换为操作指引模式。 |

### 第三层级：任务结构与交互模式（横切所有意图的“组织形态”）

| 情形分类 | 特征描述 | 系统输出组织方式 |
| :--- | :--- | :--- |
| **原子指令** | 只问“点哪里”，不问流程 | 跳过蓝图生成，直接输出单条指引，标注随之出现与消失。 |
| **线性多步流程** | 单一软件内的连续操作（如安装） | 生成锁定蓝图，步骤列表展示全部，当前步骤高亮，进度可查询。 |
| **跨应用切换流程** | 操作涉及多个窗口接力 | 蓝图中插入“窗口切换锚点”，每一步都注明“请切换到XX窗口”，并在切换后重新校准屏幕坐标。 |
| **抽象非屏任务** | 指向硬盘文件、后台进程等不可见对象 | 不调用UI解析，直接输出系统内置的通用搜索/查看指引，并明确告知“我无法直接看到内容”。 |
| **被动触发模式** | 非用户主动提问（如定时提醒、错误预防） | 仅以轻量气泡或低音量TTS呈现，不抢占全屏标注（避免干扰用户当前操作）。 |
| **主动澄清模式** | 系统反问用户的二选一/多选一问题 | 用户回答后，不重新截图，仅将答案作为新锚点填充到当前轨迹中，继续原流程。 |

### 第四层级：系统绝对边界（凡触及此三条红线，一律按标准话术拒答/降级处理）

| 红线名称 | 触及此边界的用户指令示例 | 系统的标准处理响应（非技术，纯需求层面） |
| :--- | :--- | :--- |
| **物理操作红线** | “帮我直接保存”、“自动帮我点下一步” | **必须明确拒答**：“我只能为您标注位置和提供文字步骤，请您根据指引手动操作。” |
| **个人隐私红线** | “帮我扫描硬盘找出所有照片”、“看看我的聊天记录里说了什么” | **必须明确拒答并引导**：“我无法访问您的个人文件内容。我可以指引您打开‘此电脑’并手动在搜索框输入‘.jpg’进行查找。” |
| **实时动态红线** | “帮我看这个比赛直播比分”、“跟踪这支股票的实时K线” | **必须明确降级**：“检测到屏幕内容持续刷新，我的指引可能滞后。我只能为您描述当前这1秒的静态画面内容，无法持续跟踪。” |

### 终极汇总说明

以上**9大核心意图域 × 5种指代方式 × 6种交互模式**，在扣除**3条红线拦截情形**后，构成了HAJIMI桌面指引助手**所有用户文本指令情形的完整闭合边界**。

任何不在此边界内的输入（如“帮我写个作文”“帮我发封邮件”），将被系统统一归为**“超出服务范畴”**，并回复：“我是桌面操作指引助手，无法为您生成或发送内容。如果您需要打开写作软件，我可以指引您操作。” 这一闭环覆盖了自然人机桌面交互的**全部语义空间**，无需再扩充新的根类别。
