# HAJIMI — 管理控制台设计文档 V2.1

> **版本说明**：本文档基于 V2.0 设计迭代，移除模板审核功能，新增数据流监控页面，强化“统一统计 + 监控调参”定位。管理员角色明确为 **“系统调参师”和“运行监控员”**，不再承担内容审核职责。

---

## 一、设计概述

### 1.1 设计理念

本控制台彻底转向 **SRE（站点可靠性工程）** 理念，核心目标从“管理数据”转变为“监控健康”与“调优参数”。界面以统计图表为绝对主体，操作入口高度收敛。

### 1.2 核心原则

| 原则 | 说明 |
| :--- | :--- |
| **图表优先** | 每个页面顶部均为统计图表，数据结论先于数据明细呈现。 |
| **聚合下钻** | 列表明细必须通过点击图表元素才能触发加载，避免无筛选全量查询。 |
| **滚动替代分页** | 所有列表采用无限滚动 + 总数提示，彻底移除传统的页码切换器。 |
| **导出兜底** | 所有明细区域配备“导出 CSV”按钮，满足离线深度分析需求。 |
| **实时可观测** | 关键数据流（上报、拉取、QPS）具备独立监控面板，异常可感知。 |

### 页面二：🔍 失败归因（Failure Analysis）

**定位**：专门的根因分析工作台，支持深度下钻。

#### 第一行：筛选与控制栏

| 组件 | 说明 |
| :--- | :--- |
| 时间窗口选择器 | 预设：近1小时 / 近6小时 / 近24小时 / 近7天 / 自定义（支持到秒级） |
| 失败类型下拉筛选 | 全部 / 蓝图不匹配 / LLM超时 / 解析错误 / 红线拦截 / 用户中止 |
| **“导出 CSV” 按钮** | 异步生成当前筛选条件下的全量失败记录（不受前端滚动限制） |

#### 第二行：统计图表区（双图并排）

- **左图**：失败类型分布（柱状图）
  - **交互**：点击任意柱子，下方详情列表自动过滤为该类型的失败记录。
- **右图**：失败趋势（折线图）
  - 按小时/天聚合，可切换粒度。

#### 第三行：失败详情列表（无限滚动 + 总量提示）

- **顶部横幅提示**（固定）：
  > `筛选条件：近24小时，类型：全部。共 156 条失败记录，已加载 50 条。`
- **列表字段**：
  - 失败时间 | 任务名称 | 失败步骤 | 错误摘要 | 操作（查看详情）
- **交互**：
  - 滚动到底部自动加载下一批（每次 +20 条），直至全部加载完毕。
  - 点击“查看详情”从右侧滑出详情面板（`width: 480px`）。

**详情面板内容**（满足需求：保留失败详情）：
- 任务基本信息（任务ID、用户ID、处理路径 L2/L3）。
- **预期指纹 vs 实际指纹**：并排对比展示，不一致的字段高亮标红。
- **LLM 输入快照**：折叠区域，点击展开显示 Prompt 内容（脱敏后）。
- **LLM 输出快照**：折叠区域，点击展开显示 Response 内容（脱敏后）。
- 底部操作：关闭 / 复制快照。

### 页面三：📡 数据流监控（Data Flow）—— 全新页面

**定位**：把控系统“血脉”，监控 C-S 之间的数据交换是否流畅、有无积压、有无异常断流。

#### 第一行：关键流量指标卡（4 张）

| 指标 | 示例值 | 说明 |
| :--- | :--- | :--- |
| 上行吞吐量 | 24.6 MB | 今日累计上报数据量（审计日志） |
| 下行吞吐量 | 8.2 MB | 今日累计下发数据量（配置） |
| 上报成功率 | 99.8% | 近 1 小时滑动窗口 |
| **当前队列积压估算** | **1,234 条** | 所有客户端本地待上报队列总和（关键预警指标） |

#### 第二行：数据流向拓扑图（核心视觉组件）

- 采用简化 **桑基图** 或 **关系流向图** 展示：
  `[多台客户端] ---(上报/拉取)---> [HAJIMI Server (API网关)] ---> [PostgreSQL / 外部LLM]`
- **连线标注**：
  - 每条连线上显示 **实时 QPS**（每秒请求数）和 **平均延迟（ms）**。
- **颜色编码（自动告警）**：
  - 🟢 绿色：健康（延迟 < 阈值）
  - 🟡 黄色：高负载（延迟 > 阈值 1.5 倍）
  - 🔴 红色：异常/断流（延迟 > 阈值 3 倍或连续失败）

#### 第三行：关键接口调用细项（双折线图并排）

- **左图**：`/api/audit/report` 调用趋势
  - 双轴：柱状图展示 QPS，折线图展示成功率。
- **右图**：`/api/config/pull` 调用趋势
  - 双轴：柱状图展示 QPS，折线图展示命中率。

#### 第四行：客户端版本与配置拉取分布（双饼图并排）

- **左图**：各客户端版本号占比（饼图）。
- **右图**：配置拉取频率分布（直方图）。
  - X 轴：拉取间隔区间（0-5min, 5-15min, 15-30min, >30min）。
  - 帮助判断是否有客户端未按预期间隔拉取配置（可能存在网络隔离或僵尸进程）。

#### 第五行：异常事件流（滚动列表）

- 展示**数据流层面的实时异常事件**（仅最近 20 条，无需分页）。
- 事件类型示例：
  - `[WARN] 客户端 #c12a 连续 3 次上报失败，触发指数退避重试。`
  - `[ERROR] 区域 /24 网络超时率升至 15%，触发熔断。`
  - `[INFO] 批量上报积压队列已清空，恢复正常。`
- 列表颜色区分严重级别（红/橙/蓝）。

## 四、核心交互设计（回应“面对更多数据”）

所有可能出现海量数据的场景均采用以下 **四段式组合策略**：

| 策略 | 实现方式 |
| :--- | :--- |
| **1. 聚合前置** | 必须先看图表（柱状图/饼图），**点击图表元素**才能触发下方列表加载。杜绝无筛选的全量查询。 |
| **2. 时间窗口限制** | 默认仅加载 **近 24 小时** 数据。管理员可通过时间选择器主动扩大范围（最长支持近 90 天）。 |
| **3. 无限滚动 + 进度提示** | 列表滚动加载（每次 +20 条），顶部固定显示 **“共 N 条，已加载 X 条”**，让管理员对总量心中有数。 |
| **4. 导出兜底** | 所有列表区域配备 **“导出 CSV”** 按钮。后端异步生成完整文件（无数量限制），供离线深度分析。 |
