## 问题 1：启动脚本默认路径与实际不符

**现象**：`start_all.bat` 报"系统找不到指定的路径"

**原因**：三个环境变量默认值与机器实际路径不匹配：

| 变量 | 脚本默认值 | 实际值 |
|------|-----------|--------|
| `OMNI_ROOT` | `E:\Tools\OmniParser` | `D:\ominprester` |
| `OMNI_PY` | `E:\CodingSoftwards\Anaconda\envs\omni\` | `C:\Users\86178\.conda\envs\omni\` |
| `VIDEO_RAG_PY` | conda `videorag` | conda `hajimi_ui` |

**修复**：启动前设置环境变量：
```batch
set OMNI_ROOT=D:\ominprester
set OMNI_PY=C:\Users\86178\.conda\envs\omni\python.exe
set VIDEO_RAG_PY=C:\Users\86178\.conda\envs\hajimi_ui\python.exe
```

## 问题 2：OmniParser conda 环境被 Roaming 包污染

**现象**：
```
ImportError: module 'langchain_core.runnables.base' not found
  → No module named 'et_xmlfile'
  → No module named 'jsonpatch'
  → No module named 'requests_toolbelt'
```

**原因**：`C:\Users\86178\AppData\Roaming\Python\Python312\site-packages\` 下有损坏的 langchain 系列包，conda `omni` 环境导入 paddleocr 时被污染

**修复**：修改 `HAJIMI_UI\scripts\start_omniparser.bat`，添加 `-s` 跳过用户 site-packages：

```diff
+ set "PYTHONNOUSERSITE=1"
- "%OMNI_PY%" -m omniparserserver ...
+ "%OMNI_PY%" -s -m omniparserserver ...
```

同时手动补充了 omni 环境缺失的包：
```
pip install et_xmlfile jsonpatch fastapi uvicorn starlette pydantic pydantic-core
```

## 问题 3：A-end Server 使用 Replicate 而非本地 OmniParser

**现象**：
```
A 端 HTTP 502: REPLICATE_API_TOKEN not configured
```

**原因**：实际运行的 Server 是 `HAJIMI_UI\server\`（B 端内嵌副本），其 `config.py` 默认检测后端为 `replicate_omniparser`。且 `HAJIMI_UI\server\.env` 文件不存在，全部走默认值：

```python
# HAJIMI_UI/server/config.py
DETECTOR_BACKEND: str = os.getenv("DETECTOR_BACKEND", "replicate_omniparser")  # 默认云端
OMNIPARSER_LOCAL_URL: str = os.getenv("OMNIPARSER_LOCAL_URL", "http://127.0.0.1:8000")  # 端口也不对
```

**修复**：创建 `HAJIMI_UI\server\.env`：
```env
DETECTOR_BACKEND=local_omniparser
OMNIPARSER_LOCAL_URL=http://127.0.0.1:8002
OMNIPARSER_LOCAL_TIMEOUT=360
OMNIPARSER_LOCAL_MAX_SIDE=960
```

## 问题 4：Server venv 不存在，回退到系统 Python

**现象**：
```
module 'cv2' has no attribute 'cvtColor'
```

**原因**：`start_server.bat` 先检查 `HAJIMI_UI\server\.venv\`，不存在则回退到 `python`（系统 PATH）。系统 Python 的 cv2 有问题。

**修复**：创建 `HAJIMI_UI\server\.venv` 并安装依赖：
```bash
cd HAJIMI_UI
python -m venv server\.venv
server\.venv\Scripts\python.exe -m pip install fastapi "uvicorn[standard]" pydantic pydantic-settings python-dotenv httpx loguru opencv-python sqlalchemy
```

---

## 问题 5：Server venv 缺少 Pillow 依赖

**现象**：
```
ModuleNotFoundError: No module named 'PIL'
```

**原因**：`HAJIMI_UI\server\.venv` 创建时安装了 `fastapi uvicorn pydantic pydantic-settings python-dotenv httpx loguru opencv-python sqlalchemy`，但漏了 `Pillow`（PIL）。`server/services/image_utils.py` 中 `from PIL import Image` 引用失败，导致 Server 无法启动。

**修复**：
```bash
cd HAJIMI_UI
server\.venv\Scripts\python.exe -m pip install Pillow
```

## 正确的启动方式

```batch
cd /d d:\HAJIMI\Fuzzy-Visual-Assisted-Question-Answering-System\HAJIMI_UI
set OMNI_ROOT=D:\ominprester
set OMNI_PY=C:\Users\86178\.conda\envs\omni\python.exe
set VIDEO_RAG_PY=C:\Users\86178\.conda\envs\hajimi_ui\python.exe
scripts\start_all.bat
```

启动顺序：① OmniParser (:8002) → ② A-end Server (:8010) → ③ B-end Client (GUI)

停止：关闭三个 cmd 窗口，或用 `scripts\stop_all.bat`
